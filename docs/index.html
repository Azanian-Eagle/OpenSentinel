Regulatory Compliance & Data Protection

Built for the Modern Regulatory Landscape: OpenSentinel is systematically designed from the ground up to guarantee absolute compliance with the most stringent global data protection laws, including South Africa's Protection of Personal Information Act (POPIA), Europe's General Data Protection Regulation (GDPR), and the California Consumer Privacy Act (CCPA).

  • ✅ Zero Personally Identifiable Information (PII): We categorically never collect IP addresses, browser fingerprints, email addresses, or any other data that could potentially identify an individual user.
  • ✅ Strict Data Minimisation: The sensor only measures the mathematical variance of user interactions (mouse linearity, speed, keystroke intervals). This abstract telemetry is completely anonymous and fundamentally impossible to reverse-engineer into personal profiles.
  • ✅ Localised Processing: Our state-of-the-art ONNX machine-learning model runs entirely on your own server infrastructure. We do not utilise external cloud APIs, ensuring your users' behavioural data never crosses international borders or leaves your sovereign control.
  • ✅ No Cross-Site Tracking: We fundamentally reject the use of tracking cookies, local storage beacons, or any mechanism designed to follow users across different websites. Your interactions on site A are entirely independent of your interactions on site B.

The Solution: Behavioural Sovereignty

OpenSentinel completely shifts the security paradigm from the archaic "Challenge-Response" model to an intelligent, seamless "Passive Observation" methodology. We verify humanity by meticulously analysing how you interact with a page, rather than demanding what objects you can identify in a grainy photograph.

1. Passive Telemetry

Our ultra-lightweight JavaScript sensor (coming in at under 20kb) silently observes crucial interaction metrics such as mouse linearity, movement speed variance, and keystroke flight dynamics. Because humans are inherently imperfect and variable in their movements, whilst bots operate with rigid, mathematical precision, we can accurately distinguish between the two without ever interrupting the user journey.

2. Local Analysis

We believe that privacy is a fundamental human right. OpenSentinel processes all behavioural data locally wherever possible, extracting abstract features rather than personal identifiers. We do not need to know your identity, we only need to understand the statistical "shape" of your interaction. This approach guarantees strict compliance with global privacy standards, including South Africa's POPIA and Europe's GDPR.

3. Rust Performance

The OpenSentinel backend is engineered using Rust and the Actix-web framework, a combination renowned for its blistering performance and memory safety. This robust architecture ensures exceptionally fast verification speeds, allowing the system to handle massive concurrent traffic loads effortlessly—a feat that legacy, memory-intensive systems simply cannot match.

Technical Architecture & How It Works

OpenSentinel is a sophisticated fusion of highly optimised client-side sensors, blazing-fast Rust web services, and localised machine-learning inference. Here is a deep dive into the core mechanics.

1. The Sensor Array

When a user visits your protected application, our ultra-lightweight `sensor.js` immediately begins passive observation. It hooks into standard DOM events (mousemove, keydown) and records a high-resolution, multi-dimensional array of timestamps and coordinates. This payload is heavily obfuscated and dynamically encrypted on the client side to prevent tampering or reverse-engineering by malicious actors.

2. Cryptographic Proof-of-Work

To fundamentally deter automated script kiddies and massive DDoS attempts, the client optionally computes a mathematical Proof-of-Work (PoW) challenge (e.g., finding a specific SHA-256 hash collision) before the payload is even transmitted. This significantly increases the computational cost of spamming the API without affecting legitimate human users.

3. Rust Backend & ONNX Inference

The encrypted payload is securely transmitted to your sovereign Rust backend via an Actix-web endpoint. The server decrypts the data, prevents replay attacks using strict timestamp and nonce tracking, and immediately passes the telemetry to our locally hosted ONNX machine-learning model. This Random Forest Classifier (powered by the `ort` and `ndarray` crates) calculates the statistical probability of bot behaviour in a matter of microseconds.

Step 1: Deploy the Sovereign Backend Node

Begin by deploying your own lightning-fast Rust backend. This ensures you maintain absolute control over the entire verification process and local machine-learning inference.

terminal 
git clone https://github.com/Azanian-Eagle/OpenSentinel.git
cd OpenSentinel/server
cargo build --release
cargo run --release

Configure your environment variables to join the decentralised OpenSentinel threat intelligence network.

.env 
PORT=8080
# Enable Federated Threat Sharing Protocol (Phase 3)
FEDERATION_ENABLED=true
# Comma-separated list of trusted peers on the network
TRUSTED_PEERS=https://node1.opensentinel.org,https://node2.partner.com

Step 2: Integrate the Frontend Sensor

Include our ultra-lightweight client script directly into your web application. Initialise the sensor with your primary backend endpoint and optional fallback nodes to ensure absolute high availability.

integration.html 
<script src="sensor.js"></script>
<script>
  // Initialize OpenSentinel with advanced network awareness
  OpenSentinel.init({
    // Specify your primary backend, followed by trusted fallback nodes
    endpoints: ['https://api.yourdomain.com/verify', 'https://fallback.trust-node.com/verify'],

    // Optional: Enable client-side Proof-of-Work to aggressively deter script kiddies
    enablePoW: true,

    // Optional: Register callbacks for custom behaviour
    onSuccess: function(token) {
        console.log('Human verified successfully. Proceeding with application logic.');
        // Example: Enable submit button, inject token into form
    },
    onFailure: function() {
        console.warn('Automated bot behaviour detected. Access denied.');
    }
  });
</script>
